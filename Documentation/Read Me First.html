Introduction

This is the Readme for PatchIt!, a Python 3 application intended to provide a standard yet simple way to package and install mods for the 1999 High Voltage Software PC game LEGO Racers.

Usage

First Run

Upon running PatchIt! for the first time, you will be asked for the path to your LEGO Racers installation. Simply point it to your LEGO Racers installation with your LEGO.JAM already decompressed. PatchIt! will ensure it is a valid installation, and if so, continue to the menu. You will only need to do this once, as PatchIt! is smart enough to know if you ran it once already.

Good To Know!

A vaild LEGO Racers installation is defined by the existence of three files: LEGORacers.exe, LEGO.JAM, and GolDP.dll If these three files cannot be found, your installation cannot be confirmed.

Changing your LEGO Racers Installation

If you need to change your defined LEGO Racers installation, you can do it in a minute. Just load PatchIt! , press the 's' key, and point it to your LEGO Racers installation. If it confirms a valid installation, press the 'n' key, and you're off! If it can't verify, you'll need to enter it again.

Patch Installation

To install a PatchIt! compatible mod, simply load PatchIt!, press the 'i' key, select the PiP file, and after reading the mod information, confirm the installation. PatchIt! will automatically install it into your LEGO Racers game. If you need a patch to install, jump down to the Resources section to download one!

Patch Creation

To create a PatchIt! compatible mod, load PatchIt! and press the 'c' key. Type your mod's name, version, author, and description. Point it to your modded files, and PatchIt! will compress them into a normal ZIP archive and write the details to a PiP file. As of PatchIt! 1.0.1 Stable, if Thumbs.db is found in your modded files, it will automatically deleted to avoid an error when someone installs your patch. Don't worry, Windows will automatically recreate it. ;)
Both items use your mod's name and version for the filename, and will be located in the same folder as your uncompressed modded files. Check the the PiP Format documentation (linked in the Resources section below) for complete details.


Be Careful!

A few special characters, such as a colon (:), semicolon (;), question mark (?), back-slash (\ ), and forward-slash (/)
are not permitted as filenames, and entering these into the Name: and Version: fields will have undesired effects on your computer.
They are only permitted in the Author: and Description: fields, where no harm will be done.

Support

To request a feature, submit a code patch, or download a new release of PatchIt!, please go to https://github.com/le717/PatchIt.
To submit a bug report, please go to https://github.com/le717/PatchIt/issues, state the bug, and attach TheWritingsofPatchIt.log, located at %ProgramFiles%\PatchIt\Logs. Attaching this file will ensure the bug will be found a lot easier, and a bug fix will be released much faster.
